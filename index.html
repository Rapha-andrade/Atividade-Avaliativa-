Controle de Versão

*Os sistemas de controle de versão são ferramentas de software que ajudam as equipes de software a gerenciar as alterações ao código-fonte ao longo do tempo. Como os ambientes de desenvolvimento aceleraram, os sistemas de controle de versão ajudam as equipes de software a trabalhar de forma mais rápida e inteligente. Eles são ainda mais úteis para as equipes de DevOps, pois as auxiliam a reduzir o tempo de desenvolvimento e aumentar as implementações bem-sucedidas.

Controle de Mudanças

*O controle de mudanças é um processo utilizado para lidar com solicitações de alteração em projetos e grandes iniciativas. É parte de um plano de gestão de mudanças, que define as funções para a condução das alterações em uma equipe ou empresa.

Integração Contínua

*A integração contínua é uma prática de desenvolvimento de software de DevOps em que os desenvolvedores, com frequência, juntam suas alterações de código em um repositório central. Depois disso, criações e testes são executados.
